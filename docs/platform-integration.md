# Platform integration contract

Part 2 is intentionally additive. It does not change `src/types/contracts.ts`, `/api/tutor`, or `/api/personalize`.

## Stable client boundary

`src/lib/platform/repository.ts` exports `PlatformRepository`. The current implementation is browser `localStorage`; a future backend integration replaces that implementation while preserving these methods:

- `listCoursesForTeacher(teacherId)`
- `getCourseDashboard(courseId)`
- `recordLearningEvent(event)`
- `listTeacherMaterialPatches(courseId, chapterId)`
- `saveTeacherMaterialPatch(patch)`

The platform key is `adaptive-textbook:platform`. Learner keys remain owned by the learner experience and must not be migrated as part of this work.

## Learner-to-platform events

The learner page reports only these event types: `chapter-opened`, `block-selected`, `tutor-questioned`, `quiz-answered`, and `personalization-added`.

Every event includes `courseId`, `chapterId`, `studentId`, `conceptIds`, and `occurredAt`. Quiz and tutor events may include `payload.conceptScores`, allowing the teacher dashboard to show the latest transparent learner score without calling the AI endpoints.

## Teacher material versus personal AI patches

`TeacherMaterialPatch` is deliberately separate from `ContentPatch`:

- `ContentPatch` is individual, generated by AI, and remains in learner state.
- `TeacherMaterialPatch` is course-wide, authored by a teacher, and stored by the platform repository.

Both target stable chapter block IDs and use `placement: "after"`. The reader renders teacher material with a distinct visual treatment, so it never overwrites or masquerades as a learner-specific AI patch.

## Backend replacement checklist

1. Implement `PlatformRepository` against authenticated API routes or a database.
2. Preserve the IDs and the event shapes in `src/types/platform.ts`.
3. Keep writes asynchronous and non-blocking for learner actions.
4. Return only `published` teacher material to learners.
5. Do not make the teacher dashboard call the tutor or personalization endpoints directly.
